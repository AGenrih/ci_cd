## CI/CD Workflow

1. **Dev pushes** to `dev` branch → CI runs: `lint` → `unit-tests` → `build-test`
2. **Pull Request** created from `dev` to `master`
3. **Merge to master** → CD runs: Docker build & push to Docker Hub, then updates K8s manifests in `release` branch
4. **ArgoCD** detects changes in `release` branch and auto-syncs the cluster

## Quick Start

### Local run
```bash
python3 server/application.py
```

### Docker
```bash
docker build -t devops-psu ./server --file ./server/dockerfile
docker run -d -p 8000:8000 devops-psu
```

### Kubernetes (Minikube)
```bash
kubectl create namespace devops-psu
kubectl apply -f server-k8s-manifests/devops-psu.yml
minikube tunnel --bind-address 10.0.2.15
```

### ArgoCD Setup
```bash
kubectl create namespace argocd
kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
# Get admin password
kubectl -n argocd get secret argocd-initial-admin-secret --template={{.data.password}} | base64 -d
# Expose service
kubectl patch svc argocd-server -n argocd -p '{"spec": {"type": "LoadBalancer"}}'
kubectl patch svc argocd-server -n argocd -p '{"spec": { "externalIPs": [ "10.0.2.15" ]}}'
```

ArgoCD UI at `https://10.0.2.15:443` (login: `admin`, get password above).
